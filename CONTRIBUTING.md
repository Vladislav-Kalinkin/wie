# Contributing Rules

If you want to help this project and offer your own improvements, corrections or new solutions, be sure to read this small set of rules that will help you follow the guidelines of WIE

### Rules:

1. **It should be checked that the Pre-PR Checklist is clean** (watch README). If there are clippy warnings, test errors, formatting problems - it is highly desirable to fix

2. **Integration Suite** - You should make sure that after your changes that all test exe files pass tests and the speed of execution has not decreased. If something is broken - be sure to fix

3. **unsafe** are allowed only in critical places (for example, wie-cpu). You should make sure that after your changes the code was the minimum number of unsafe code.

4. **allow elements** (for example, too-many-arguments) should be minimal. In most cases, you can do without them.

After all the stages have been completed, you can safely send your pull request.

### Using AI in PR:

You are allowed to use the code generated by the neural network for Pull Request, but under an important conditions.

1. The code had to be checked **by you**
2. The code complies with the rules
